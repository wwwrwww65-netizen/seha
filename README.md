# نظام صحة - منصة إدارة واستعلام الإجازات المرضية 🏥🩺

نظام متكامل واحترافي مقسم إلى واجهتين (واجهة استعلام عامة للمرضى، ولوحة تحكم إدارية للموظفين)، مبني وفق أحدث معايير الأمان باستخدام **(HTML/JS/CSS)** للواجهات الأمامية، و **(PHP + MySQL)** للخادم الخلفي وقواعد البيانات.

---

## ✨ المميزات الأساسية (Core Features)

- **واجهة استعلام سريعة (Public Inquiry):** تتيح للمرضى التحقق من حالة تقارير إجازاتهم المرضية برقم الهوية ورمز الخدمة.
- **نظام تحكم بالصلاحيات (RBAC - Role Based Access Control):** دقة متناهية في الصلاحيات (مدير نظام مقابل مدخل بيانات)، وتخصيص صلاحيات (إضافة، تعديل، حذف، ولوج للوحة التحكم) لكل مستخدم بشكل مستقل.
- **إدارة كاملة للإجازات (Leave Management):** تسجيل البيانات بتفاصيلها (اسم المريض، الطبيب المالج، المسمى الوظيفي، فترات الإجازة، وبيانات المرافق إن وجدت).
- **تصميم متجاوب وحديث (Responsive UI/UX):** يعمل بسلاسة على الهواتف المحمولة وكافة الشاشات.
- **حسابات وتفعيل (Account Status):** إمكانية إيقاف حسابات المستخدمين مؤقتًا (Active/Inactive Toggle) ومنعهم من تسجيل الدخول بحال تم إيقافهم.
- **ميزة "الإخفاء الذكي" للبيانات الاختيارية:** مثل (المرافق وصلة القرابة) لا تظهر في الاستعلام إلا إذا تمت تعبئتها بواسطة المدخل المعتمد.

## 🛡️ الحماية والأمان (Security & Hardening)

تم بناء النظام ليكون جاهز للإطلاق المباشر بأعلى معايير الحماية (Production Ready):
1. **تشفير كلمات المرور (Password Hashing):** باستخدام خوارزميات التشفير القياسية (Bcrypt with Cost=12) عبر دوال PHP. لا تُخزن مطلقاً كنص عادي.
2. **منع حقن قواعد البيانات (SQL Injection Protection):** استخدام معماريّة `Prepared Statements` و `PDO` مع تعطيل `Emulate Prepares`.
3. **حماية الجلسات (Session Fixation & Hijacking):** يتم استخدام `session_regenerate_id()` عند كل تسجيل دخول، بالإضافة إلى تأمين الكوكيز بخصائص (`Secure`, `HttpOnly`, `SameSite=Strict`).
4. **جدار الحماية المصغر (API Gateway):** كل نقطة وصول (Endpoint) محمية بـ `requireAuth()` و `requirePermission()` لضمان استحالة الاستعلام أو الحذف باختراق الواجهات.
5. **منع الاختراقات المشتركة (XSS & Clickjacking):** تدعيم التجاوب العام بـ `Security Headers` مثل `X-Content-Type-Options`, `X-Frame-Options`, `X-XSS-Protection`.
6. **حماية المجلدات والملفات (Directory Traversal):** ملفات `.htaccess` مخصصة تمنع استعراض المجلدات (`Options -Indexes`) وتخفي الملفات الحساسة كملفات الـ `SQL`.

## 🚀 طريقة التثبيت والرفع (Installation - Hostinger / cPanel)

### الخطوة الأولى: تجهيز قاعدة البيانات
1. ادخل إلى لوحة تحكم الاستضافة (مثل Hostinger hPanel أو cPanel).
2. أنشئ قاعدة بيانات MySQL جديدة ومستخدم جديد، واربطهما وضع كلمة سر معقدة.
3. ادخل إلى **phpMyAdmin**، اختر القاعدة الجديدة، واضغط على **Import (استيراد)**.
4. ارفع الملف المرفق مع النظام: `database_schema.sql`.

### الخطوة الثانية: ربط الخادم (Configuration)
افتح الملف `api/config.php` بأي محرر نصوص، وقم بتغيير الإعدادات التالية لتطابق البيانات التي أنشأتها في الخطوة 1:
```php
$host = "localhost"; // او حسب الاستضافة
$db_name = "your_hostinger_db_name";
$username = "your_hostinger_db_user";
$password = "your_hostinger_db_password";
```

### الخطوة الثالثة: الرفع الجاهز المباشر (Upload to Web Root)
1. افتح **File Manager** وانتقل إلى مجلد `public_html`.
2. ارفع جميع ملفات المشروع: مجلد `api` بملفاته، والواجهات (`index.html`, `dashboard.html` ... الخ)، وكل الصور وملف الستايل `style.css`.
3. قم بتفعيل الحماية في السيرفر (لو رغبت) من الملف `.htaccess` وإلغاء التعليق عن التوجيه الإجباري للـ HTTPS.

## 📂 هيكلة النظام (Directory Structure)

```text
/seha
├── index.html              # صفحة الاستعلام العامة للمرضى
├── login.html              # نظام تسجيل الدخول للموظفين والمدراء
├── dashboard.html          # لوحة التحكم وإدارة الإجازات (الجدول المتقدم والإعدادات)
├── issue-leave.html        # صفحة إصدار تقرير إجازة مرضيّة جديد/تعديل
├── style.css               # الأنماط الجمالية للواجهة متجاوبة بالكامل
├── db.js                   # محرك الواجهة الأمامية المسؤول عن الـ Fetch Promises
├── database_schema.sql     # هيكل بناء قاعدة البيانات الجاهز
├── .htaccess               # إعدادات حماية السيرفر وملفات الكاش
└── api /                   # مجلد خدمات السيرفر الآمنة (Backend)
    ├── .htaccess           # منع وصول المتصفح المباشر لملفات الـ PHP
    ├── config.php          # ملف الربط ودوال الحماية والجلسات
    ├── auth.php            # نظام عمليات تسجيل الدخول والخروج والتحقق
    ├── inquiry.php         # خدمة البحث العامة للمرضى
    ├── leaves.php          # خدمة العمليات (CRUD) الخاصة بالإجازات
    └── users.php           # خدمة العمليات للمستخدمين والصلاحيات
```

تم التصميم والتطوير بنجاح! 
أي احتياجات للصيانة، يمكن التوجه لملفات السيرفر داخل `api` ولملفات الواجهات في المسار الجذر.
